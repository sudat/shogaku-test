# 算数テスト追加 設計メモ

## 目的

既存の漢字テストに加えて、学年別の算数テストを追加する。漢字テストは4択を維持し、算数テストは数字の直接入力で回答する。

この設計では、既存の漢字クイズを大きく作り替えず、最初に教科を選ぶ入口を追加する。算数クイズは新しい機能領域として実装し、画面の流れと結果表示の考え方を既存機能に揃える。

適用する設計原則:

- KISS: 既存の10問テスト構成に合わせ、算数もシンプルな開始、回答、結果の流れにする。
- YAGNI: 初期実装では「足し算だけ」「引き算だけ」などの細かいモード分けを作らない。
- DRY: 履歴保存や結果表示の考え方は既存と揃える。ただし、早すぎる共通化は避ける。

## 画面構成

`src/App.jsx` に教科選択画面を追加する。最初の画面で「かんじ」と「さんすう」を選べるようにする。

- 「かんじ」を選ぶと、既存の漢字テスト画面へ進む。
- 「さんすう」を選ぶと、新しい算数テスト画面へ進む。
- どちらのテスト画面にも、教科選択へ戻る導線を置く。

漢字テスト側は、現在の「学年選択」「読み/書き選択」「10問クイズ」「結果」の流れを維持する。算数テスト側は、学年選択後にすぐ開始する。算数には読み/書きに相当するモードを置かない。

## 算数テスト仕様

算数テストは10問制にする。問題は開始時にランダム生成する。

学年別の出題範囲:

| 学年 | 出題内容 |
| --- | --- |
| 1年生 | 一桁の足し算、一桁の引き算 |
| 2年生 | 二桁の足し算、二桁の引き算、一桁の掛け算 |
| 3年生 | 二桁 × 一桁の掛け算 |

引き算では、答えがマイナスになる問題を出さない。繰り上がりと繰り下がりは許可する。

2年生の問題は、足し算、引き算、掛け算をだいたい均等に出す。10問は3種類で割り切れないため、演算種別のリストを作ってシャッフルし、3問から4問ずつになるようにする。

## 回答と判定

算数テストは数字を直接入力する。回答欄は数値入力を想定し、スマートフォンでは数字キーボードが出る指定にする。

回答時の動き:

- 空欄では判定しない。
- 「こたえる」ボタン、または Enter キーで判定する。
- 正解なら「せいかい！」を表示する。
- 不正解なら正しい答えを表示する。
- 判定後は入力をロックし、短い待ち時間のあと次の問題へ進む。
- 10問終了後に結果画面へ進む。

不正解時の答え直しは行わない。既存の漢字クイズに近いテンポを優先する。

## データ構造

算数問題は固定データではなく、ロジックで生成する。

想定する問題データ:

```js
{
  left: 24,
  operator: "×",
  right: 3,
  answer: 72
}
```

表示では `24 × 3 = ?` のように組み立てる。判定では入力値を数値化し、`answer` と比較する。

## 実装方針

新しく `src/features/math-quiz/` を追加する。

想定ファイル:

- `MathQuiz.jsx`
- `logic.js`
- `components/StartScreen.jsx`
- `components/QuizScreen.jsx`
- `components/ResultScreen.jsx`

既存の `src/features/kanji-quiz/` は、算数追加のために大きく変更しない。必要な変更は、教科選択へ戻るためのプロパティ追加に留める。

履歴保存は既存の仕組みを参考にし、漢字と算数の記録が混ざらないキーを使う。例: `shogaku-history-${grade}-math`。

## 検証方針

ロジックテストでは、問題生成の条件を確認する。

- 1年生の引き算でマイナス答えが出ない。
- 1年生は一桁の足し算と引き算だけを出す。
- 2年生は二桁の足し算、二桁の引き算、一桁の掛け算をだいたい均等に出す。
- 2年生の引き算でマイナス答えが出ない。
- 3年生は二桁 × 一桁の掛け算だけを出す。

UIテストでは、主要導線を確認する。

- 教科選択から漢字テストへ進める。
- 教科選択から算数テストへ進める。
- 算数テストで数字を入力し、正解または不正解の表示が出る。
- 10問終了後に結果画面へ進む。

UI変更後は、ローカル表示で主要操作、レスポンシブ表示、console error の有無を確認する。
